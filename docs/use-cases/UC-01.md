# Use Case UC-01: Browse / Search Products and Add to Cart

**Primary Actor:** Customer (Shopper)  
**Scope:** NovaStore – Niche E-commerce Prototype  
**Level:** User Goal

---

## Brief
The customer browses or searches the product catalog, views product details, and adds an item to their shopping cart.

---

## Stakeholders
- **Customer:** Wants to easily discover and purchase products.
- **Store Admin:** Wants products to be discoverable and purchasable.

---

## Preconditions
- Product catalog exists in the database (or seeded data).
- System and API are online.
- A cart identifier exists (guest cart or user session).

---

## Trigger
The customer visits the store homepage or performs a search.

---

## Basic Flow
1. Customer opens the browse page.
2. System requests product data from the API.
3. Products are displayed to the customer.
4. Customer searches or filters products.
5. System returns filtered results.
6. Customer selects a product.
7. System displays product details.
8. Customer clicks **Add to Cart**.
9. System sends request to API to update the cart.
10. System confirms item was added.

---

## Alternate Flows
- **No results:** System displays a “No products found” message.
- **Out of stock:** API returns an error; item cannot be added.
- **Invalid product ID:** System displays an error message.

---

## Postconditions
- Cart contains the selected item.
- Cart totals can be calculated from stored data.
